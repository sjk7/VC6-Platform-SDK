Microsoft UDDI SDK Release Notes

Component Registration

You must register the following components on your computer before you can use the Microsoft UDDI SDK:

    • Microsoft.Uddi assembly – this library contains the implementation of the UDDI support for linking with .NET Framework programs that use UDDI
    • UDDI Publish Wizard – this wizard publishes details about a Web service and its associated Web Service Description Language (WSDL) file in a UDDI site

These components are included in the Microsoft Platform SDK and installed by default to %ProgramFiles%\Microsoft SDK\Bin\. They can be registered using the UddiSdkRegister utility. This utility adds the Microsoft.Uddi assembly to the .NET Framework Global Assembly Cache and to the Visual Studio .NET components list and add the UDDI Publish Wizard to the Tools menu within the Visual Studio .NET development environment. To register these components, perfomr the following steps:

    1. Click the Start button, click Run
    2. Type "%ProgramFiles%\Microsoft SDK\Bin\UddiSdkRegister"

Documentation Errors and Changes

The Microsoft UDDI SDK documentation is included in the Platform SDK help files. The documentation contains the following errors:

    • The topic "Authenticating with a UDDI Server" contains incorrect URLs. The correct URLs are:
        http://mySite/uddi/inquire.asmx
        https://mySite/uddi/publish.asmx
        http://mySite/uddi/extension.asmx

The following late-breaking changes did not make it into the documentation:

    • Uddi.Name – the documentation incorrectly lists this as a structure. It is a class.
    • Uddi.ManagedUrl.AdvanceSelectedIndex – the documentation incorrectly lists this method as protected. The method is private and unavailable to application developers.
    • Uddi.TModels.CommonCanonical - the NtisGovNaics2002 field contains the predefined tModelKey that specifies the NAICS 2002 business categorization scheme.
    • Services.BusinessService.BusinessService(String) and Services.BusinessService.BusinessService(Name) - these BusinessService constructors refer to the String and Name arguments as the name of the business. These argument descriptions are incorrect. The String and Name arguments contain the name of the service.
