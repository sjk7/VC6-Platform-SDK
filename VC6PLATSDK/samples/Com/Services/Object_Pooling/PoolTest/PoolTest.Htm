Object Pooling

Overview

This Object Pooling sample is a very simple one that shows how to utilize the COM+ object pooling. The basis for this sample is a logging object, which opens a specified file on construction and then shares access to this file via the pooling ability of COM+. It closes the file on destruction.

Project Files

Subdirectory Description
PoolTest Main dialog box that periodically calls the LogWriter object.
PoolTest\LogWriter COM+ object that actually writes the information to the file

Build Notes

Note:
Both the LogWriter object and the PoolTest client must be built in Unicode.

  1. Build the PoolTest client and the LogWriter object by opening the PoolTest.dsw workspace from Visual C++ and clicking Build/Rebuild All a the menus. You can also use the Microsoft NMAKE-compatible makefile (MAKEFILE).
  2. Create a new empty COM+ Server application.
  3. Add the LogWriter.dll as a new component to the application.
  4. On the Activation tab of the properties for the LogWriter component, click the Enable Object Pooling option.
  5. Set the Minimum pool size to zero (0) and the Maximum pool size to one (1).
  6. Select the Enable object construction check box and type the name of the file to receive the log entries in the Constructor string text box.
  7. All other property defaults are acceptable.

Run Notes

  1. To run this sample, start multiple instances of the PoolTest.exe program.
  2. At the rate of once every second, an entry will be written to the file specified in Component Services. After each instance has written several entries, close each of the PoolTest clients.
  3. Open the logging file and note that the entries from each of the ProcIDs are interweaved, as the LogWriter object was served up to each client from the pool.
