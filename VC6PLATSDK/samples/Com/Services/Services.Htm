Component Services

Overview

Component Services (COM+) makes it easier to develop, deploy, and administer component-based applications. With COM+ you can also leverage current investments in COM and Microsoft Transaction Server (MTS).

New services include the following:

  • Queuing -- unified objects and messaging
  • Events -- unbound methods
COM+ contains improved services including the following:
  • Transactions, Servers, Security, Administration
  • Interception
  • Unified COM and MTS for single programming/admin model
  • Interoperability (XA, TIP, TxF, CRM)
  • Backward compatibility -- 100% compatible with existing COM and MTS components, with same or better performance

COM+ offers developers the following benefits:

  • Simplification through reduction
  • Reduction in the number of APIs required
  • Increase in abstraction
  • Zero plumbing
  • Simple programming model (clients create, use, release and servers do work)
  • Strong encapsulation

To learn how to use COM+, you will find sample applications demonstrating the following features:

  • BYOT
  • Concurrency
  • Compensating Resource Manager
  • Events
  • Object Pooling
  • Queued Components
  • Resource Dispensers
  • Security
  • Transactions

Project Directories

Directory Name Description

CRM Several examples demonstrating how and why to use Compensating Resource Manager (CRM).

Object_Pooling Sample demonstrating object pooling and construction.
Queued_Components Sample demonstrating how to implement components that can be queued.
Resource_Dispenser Sample demonstrating how to build a resource dispenser.
