Build Notes

The Queued Components samples provide both a generic makefile as well as a Microsoft Visual Studio project file. The generic makefile is intended to support command-line compilation in the context of the installed Microsoft Platform SDK.

The supplied Platform SDK makefile (makefile) for the Queued Components samples are Microsoft NMAKE-compatible. To create a debug build of the sample, simply issue the NMAKE command in the command prompt window from the sample's main directory (QCSamp2).

Note:     The makefile assumes you have installed the Platform SDK and set the system environment paths as required for compilation with the Platform SDK.

If you want, you may choose to use Visual Studio to build the sample. A workspace file is provided for this purpose. For example, to load all the projects for QCSamp2, you can run Visual Studio at the command prompt in the QCSamp2 directory by typing the following command:

c:\> msdev QCSamp2.dsw

You can also just double-click the QCSamp2.dsw file in Windows Explorer to load the sample's workspace into Visual Studio. From within Visual Studio you can then browse the C++ classes of the sample's source and generally perform the other edit-compile-debug operations provided by Visual Studio.

Note:     As part of the Platform SDK, the compilation of these samples from within Visual Studio requires the proper setting of the Platform SDK directory paths within Visual Studio. See the Platform SDK installation notes.
