Running QCSamp1 and QCSamp2 on multiple computers

In this procedure, you will distribute the queued component sample applications over several computers. When an application calls a COM+ application on a remote computer, it does so via an application proxy.

Running QCSamp1

  1. On computer A, create and configure the COM+ application QCOrder per the instructions documented in Running the sample.
  2. On computer B, create and configure the COM+ application QCShip.
  3. On computer C, create and configure the COM+ application QCNotify.
  4. On computer B, export an application proxy for QCShip.
  5. On computer C, export an application proxy for QCNotify.
  6. On computer A, import the application proxy created in step 4.
  7. On computer B, import the application export created in step 5.
  8. On computer A, run driver.exe.
  9. Experiment with running this configuration with QCShip and/or QCNotify not running.

Running QCSamp2

  1. On computer A, create and configure the COM+ application QCOrder per the instructions documented in Running the sample.
  2. On computer B, create and configure the COM+ application QCShip.
  3. On computer C, create and configure the COM+ application QCNotify.
  4. On computer B, export an application proxy for QCShip2 (see below).
  5. On computer C, export an application proxy for QCNotify.
  6. On computer A, import the application proxies created in steps 4 and 5.
  7. On computer A, run driver.exe.
  8. Experiment with running this configuration with QCShip2 and/or QCNotify not running.

To export an application proxy

The following procedure will create an .MSI file with the specified file name. This file should be made available to the computer that is going to import and use the COM+ application.

  1. Start Component Services Explorer by clicking Start, point to Programs, point to Administrative Tools and then click Component Services.
  2. Select Console Root, click Component Services, click Computers, click My Computer, and then click COM+ Applications.
  3. You will see the icons for the COM+ applications, for example, QCShip.
  4. Right-click the application and select Export. This will start the COM Application Export Wizard
    5. . Click Next.
  5. Enter the full path for the application file (.MSI file) you want to create.
  6. Click Export as an Application Proxy and then click Next.
  7. Click Finish.

To import an application proxy

  1. Start Component Services Explorer by clicking Start, point to Programs, point to Administrative Tools and then click Component Services.
  2. Select Console Root, click Component Services, click Computers, click My Computer, and then click COM+ Applications.
  3. Right-click the COM+ Applications folder, click New, and then click Application. This will start the COM Application Install Wizard.
  4. Click Next.
  5. Click Install pre-built application(s).
  6. Click the .MSI file created by exporting the application (see To export an application proxy, above).
  7. Click Open.
  8. Click Next. This will install the application into the directory shown.
  9. Click Next.
  10. Click Finish.
