QCSamp3

Build Notes

This sample can be built by using VBBuild.Bat. Ensure during the build that project references are correct.

Note:     The Visual Basic debugging environment intercepts component activation. You should not create a project group that combines the components and then run the driver program with the project group open.

Running the sample

Click here for detailed instructions on how to set up and run QCSamp3.

Note: Unlike defining an interface via IDL, the interfaces defined in Visual Basic will not be marked as queued when the components are installed. You must use Component Services explorer to mark the interfaces as queued. Also note that the default names for interfaces defined in Visual Basic are prefixed with an "_". The interfaces to mark as queued for the QCSamp3 application are _INotify and _IShip.

Running on multiple computers

Click here for detailed instructions on how to set up and run QCSamp3 on multiple computers. It is in this configuration where you can begin to see the potential for building disconnected applications with Queued Components.

Optional ADO DSNs

If SQL Server or some other Microsoft Distributed Transaction Coordinator-compatible (MS DTC) Data Source Name (DSN) is available, then you can run the sample in a configuration that also updates a database while performing Queued Component operations. In this configuration, MS DTC is automatically enlisted to ensure that operations against the database and Message Queuing (MSMQ) are performed atomically by using the two-phase commit protocol.

ADO connection to the DSNs specified on the form as user "sa" with password "". The default is to use the same database for both order and ship records by using the FILEDSN "QCSample". This is configurable using the input box on the driver form.

Configure ADO DSNs

The easiest way to support this configuration is by having SQL Server available on the network and by configuring the appropriate DSNs and user accounts. Driver.exe will automatically create the necessary tables.

Warning:     If using SQL Server, configure SQL Server and the DSNs to use TCP/IP instead of named pipes.
