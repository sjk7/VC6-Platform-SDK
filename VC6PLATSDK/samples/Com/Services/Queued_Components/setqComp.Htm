Confirm the Queued property for the interfaces on the QCShip and QCNotify components

In this procedure you will confirm the Queued property for the interfaces on the QCShip and QCNotify components.

  1. Start the Component Services Explorer. Click Start, point to Programs, point to Administrative Tools, and then click Component Services.
  2. Select Console Root, click Component Services, click Computers, click My Computer, click COM+ Applications, click QCShip, click Components, click QCShip.Ship.1, and then click Interfaces.
  3. Right-click IShip interface and then select Properties.
  4. Click Queuing tab.
  5. Confirm that the Queued option is clicked.
  6. Repeat this procedure for INotify interface on the QCNotify.Notify.1 component in the QCNotify COM+ application.

These interfaces were automatically set to Queued during component install because the interfaces were marked QUEUEABLE in the idl file that was used to define these interfaces. See QCShip.idl and QCNotify.idl.
