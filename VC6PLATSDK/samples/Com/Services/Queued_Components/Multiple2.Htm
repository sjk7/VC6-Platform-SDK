Running QCSamp2 on multiple computers

This procedure distributes the queued component sample application QCSamp2 over several computers. When an application calls a COM+ application on a remote computer, it does so via an application proxy.

To create and install an application proxy

  1. On computer A, create and configure the COM+ application QCOrder per the instructions documented in Running the sample.
  2. On computer B, create and configure the COM+ application QCShip.
  3. On computer C, create and configure the COM+ application QCNotify.
  4. On computer B, export an application proxy for QCShip.
  5. On computer C, export an application proxy for QCNotify.
  6. On computer A, import the application proxy for QCShip and QCNotify created in steps 4 and 5.
  7. On computer B, import the application proxy for QCNotify created in step 5.
  8. On computer A, run driver.exe.
  9. Experiment with running this configuration with QCShip and/or QCNotify not running.
