DataAcADO

Overview

The Data Access sample shows how to write data access components that participate in transactions that use three different data access technologies: RDO, ADO, And OLE DB. 

This sample is a simple COM+ component that obtains a first name from SQL Server’s pubs database and returns this name to the COM+ client.

Creating Your Own Project

To recreate this sample from scratch

1. Use the ATL COM App Wizard to create a new DLL. Click Support MTS.

2. Insert a new ATL COM object.  Click MS Transaction Server type. On the MTS tab, click Support IObjectControl and Can be Pooled.

(In this example, the object is named "ADOComponent.")

3. In ClassView, right-click the object's interface and click Add  method.  Type the method name, and in the Parameters field, type:

   [in, out] VARIANT * returnval

4. Use ClassView to open the .cpp file for your method.  (In this case, the file is named  DOComponent.cpp.)

5. Below the includes at the top of the cpp file, insert the following line:

   #import "msado15.dll" no_namespace rename("EOF", "adoEOF" )

6. On the Build menu, click Set Active Configuration and then click Win32 Debug  configuration. (In a production environment, you should Release Min Size for COM+ components.)

7. Under Project Settings, select the C/C++ tab and add the /GX flag to enable C++ exception handling.  (C++ exception handling is required by ATL smart pointers.)  Also, delete /D "_ATL_MIN_CRT".

8. Insert the following code from this and the following page in your method and then compile:

   _RecordsetPtr adoRs;

 

   _bstr_t Source("select au_fname from authors where au_lname = 'White'");

   _bstr_t Connect("DSN=pubs;Uid=sa;Pwd=;Database=pubs");

 

   try

   {

      _variant_t InVar;

 

      adoRs.CreateInstance(__uuidof(Recordset));

      adoRs->Open( Source,

                Connect,

                adOpenForwardOnly,

                        adLockReadOnly,

                        adCmdText);

      InVar = adoRs->Fields->GetItem("au_fname")->Value;

      VariantClear(returnval);

      VariantCopy(returnval, &(InVar.Detach()));

 

      adoRs->Close();

   }

   

   catch(_com_error z)

   {

      if(adoRs)

            {

         adoRs->Close();

         adoRs = NULL;

            }

   

            if (m_spObjectContext)

                  m_spObjectContext->SetAbort();

            return z.Error();

      }

 

   if (m_spObjectContext)

   {

            m_spObjectContext->SetComplete();

   }

 

   adoRs = NULL;

   return S_OK;

 

Remarks

In order for this sample to work, a system DSN named pubs should be present, and this data source should point to an instance of SQL Server’s pubs database.
