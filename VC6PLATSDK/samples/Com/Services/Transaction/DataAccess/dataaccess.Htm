DataAccess

Overview

The Data Access sample shows how to write data access components that participate in transactions that use three different data access technologies: Remote Data Objects (RDO), ActiveX Data Objects (ADO), and OLE DB. Each subdirectory contains a simple COM+ component that obtains a first name from SQL Servers pubs database and returns this name to the client.

Project Files

Subdirectory

Description

DataAcADO

Demonstrates how to use ADO and ATL in a COM+ application. A simple COM+ component obtains a first name from SQL Server's pubs database by using ADO and returns this name to the client.

DataAcOLEDB

Demonstrates how to use ADO in an OLE DB application. A simple COM+ component obtains a first name from SQL Server's pubs database by using OLE DB and returns this name to the client.

DataAcRDO

Demonstrates how to use RDO in a COM+ application. A simple COM+ component obtains a first name from SQL Server's pubs database by using RDO and returns this name to the client.

DataAcTest

Simple Visual Basic form that performs the same simple query in four ways:
1) Calling ADO directly from Microsoft Visual Basic
2) Using the ADO component
3) Using the RDO component
4) Using the OLE DB component.

InstallDataAccess.vbs

A script that creates a Data Access application and installs the three DataAccess components.

Build Notes

Build each of the samples by using the respective tool and create the COM+ application by using the InstallDataAccess script.

Remarks

In order for these samples to work, a system DSN named pubs should be present, and this data source should point to an instance of SQL Server’s pubs database.
