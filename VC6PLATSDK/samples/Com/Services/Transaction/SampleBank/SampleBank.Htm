Transaction Sample Bank

Overview

The Sample Bank application consists of a simple Microsoft Visual Basic (VB) form through which a user can perform basic bank transaction such as debits, credits, and transfers. VB performs these action by instantiating COM+ objects to make database updates using ADO. These COM+ objects are implemented in three different programming languages and through the UI the user can select which version or component to use.

The components can be installed as standard COM objects or as COM+ objects.If used as standard COM objects, the components do not participate in transactions. BankAdmin contains a sample script that uses the COMAdmin scripting components to create a COM+ application and to register the components.

Project Files

Subdirectory

Description

Visual J++ implementation of Sample Bank components. Demonstrates how to participate in a transaction.

Sample scripts demonstrating how to automate the installation of COM+ applications.  For more information about the sample scripts, see Admin.htm

VBBank

Visual Basic client for the Sample Bank application.

Build Notes

  • VC: Either run nmake -a from the vbbank\ folder or run and build Transaction.dsw fron the Transaction\ folder.
  • VB: Run and build account.vbp from the vbacct\ folder and bank.vbp from the vbbank\ folder.
  • VJ: Ensure that the CLASSPATH user environment variable has the path to the \Transaction\SampleBank\vjacct folder listed first then run and build Vjacct.vjp from the vjacct\ folder.

Remarks

These samples require that a FILESDSN called BankSample be installed and correctly configured. This DSN should have been installed by the Platform SDK Installer. If BankSample.dsn does not exist, create a FILEDSN that points to the pubs database on any valid SQL Server.

The subdirectories step1 through step8 in the vbacct directory demonstrate one possible development path for the sample code.The final version of the sample code is contained in vbacct directory itself.
