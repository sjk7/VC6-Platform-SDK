Security Tester

Overview

Microsoft Visual Basic application using Security Sample methods


To Build

  1. Ensure that the Security Sample's Microsoft Visual C++, Visual Basic, and Java DLLs are all built.

  2. Open ServicesSecurityTester.vbp.

  3. Ensure that all references are resolved.

  4. Build ServicesSecurityTester.exe.


To Configure

  1. In Component Services Explorer create a new package named Security Tester.

  2. Right-Click the new COM+ application, Security Tester.

  3. In Properties, click the Security Tab, click Authorization, and then selec Enforce access checks for this application check box.

  4. Click OK.

  5. Under the Security Tester COM+ application, create an Administrator role (beware of case sensativity).

  6. Add an intended User to Administrator role.

  7. Install the VC, VB, and VJ Security Sample DLLs.

  8. For each component, in Properties, click the Security Tab, and select the Administrator Check box.


To Run

  1. Run ServicesSecurityTester.exe.

  2. Click Who is logged on? to bring up up a message box displaying the answer.

  3. Click Are you an Administrator? to see if you are listed in the Administrator Role of the Services_Security package.

  4. Change Users in the Administrator Role or Security Options with the Security Tester application, to manipulate values returned by the security methods.
